rebuilds `buildsystem-core` toward the from-scratch design in
[000-target-architecture.md](000-target-architecture.md) using a strangler
pattern: the plugin compiles and behaves identically after every plan.
`buildsystem-api` stays binary-compatible throughout (deprecate-only, never
delete).

starting, honor its STOP conditions, run every verification gate, and update your

## Dependency notes

- **001 before everything**: zero tests exist at `67beca7`; the harness is the
  only verification beyond `compileJava`.
- **002 before all refactors**: known bugs are fixed in place so the rebuild ports
  *correct* behavior (and so behavior changes are reviewable in isolation).
- **003/004 are the leaf dependencies**: nearly every class reads config or sends
  messages; their receiver migrations establish the transitional access rule
  (`plugin.getX()` / `// TODO(plan-014): inject` markers) the later plans rely on.
- **005 → 006** and **007 → 008** are framework-then-migration pairs.
- **009 after 008** is a recommendation, not a hard dependency (008 deletes
  `InventoryListener`, shrinking 009's listener inventory).
- **014 is the cleanup gate**: it consumes the TODO markers from 003/004 and
  deletes `BuildSystemPlugin.get()`; running it earlier than 010/011/013 means
  re-doing work.
- **Parallelization**: 012 is independent of 005–011 (only needs 002/003) and can
  run any time after them. 005/006 and 007/008 pairs can run in parallel with each
  other and with 010/011. 017 follows 010 directly; 018 runs late (it consumes
  009's registrar and 014's injection).
- **016 runs last, always** (despite its file number — numbering reflects when it
  was authored, the table above is the execution order). It closes the series:
  package conformance, modernization sweep over untouched files, docs.

## Findings considered and rejected

(So nobody re-audits these.)

- **`BuilderImpl` "implementation leak" in the API module**: false alarm — it is
  package-private behind a sealed interface; a legitimate pattern. (Plan 015 still
  converts it to a record, as polish.)
- **Convert `SettingsImpl` to a record**: rejected — player settings are mutable by
  design; records don't fit.
- **Adventure/Component chat API migration**: rejected — the plugin targets
  spigot-api (not Paper) with legacy `&` codes throughout `messages.yml` and a
  custom hex/gradient engine (`util/color`). Migration would be high-risk,
  user-visible, and out of proportion to the benefit.
- **MockBukkit / server-mocking test framework**: rejected for now — version
  alignment against spigot-api 26.1.2 is uncertain; pure-logic tests + extraction
  of testable seams gives most of the value without the dependency risk.
- **Backup retry/exponential backoff**: rejected — changes user-facing timing for
  an unproven need; revisit on operator demand (noted in plan 012).
- **`/physics all` colliding with a world named "all"** (`PhysicsCommand:68`):
  acknowledged quirk, deliberately preserved (behavior change without user demand).
- **Make the WorldEdit blocked-command list user-configurable**: rejected — it is
  integration knowledge, not configuration; it moves to
  `integration/worldedit/WorldEditCommands` instead (plan 009).
- **Quit-listener cleanup for `PaginatedInventory.invIndex` leak**: rejected as
  throwaway — the leak dies by design in plan 007/008 (per-open menu instances).
- **`UpdateChecker` rewrite**: kept minimal — it is attributed third-party code;
  plan 014 only de-singletons it.

## Known facts for executors

- Build: `./gradlew :buildsystem-core:compileJava` (gate), `./gradlew build`
  (full, incl. shadowJar). Tests (after 001): `./gradlew :buildsystem-core:test`.
- Java 25 toolchain via `buildSrc`; version catalog at `gradle/libs.versions.toml`.
- Conventional commits (`fix:`, `refactor:`, `test:`, `docs:`, `chore:`).
- License header: 17-line GPL block at the top of every `.java` file — copy from
  any existing file.
- The maintainer's comment policy: no comments that restate code; useful Javadoc
  on all public API; internal comments only for non-obvious decisions.
